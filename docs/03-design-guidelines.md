# Design Guidelines

## Goal

Create a playful, colorful and polished visual experience.

Ordboble should feel:
- joyful
- alive
- creative
- lightweight
- modern
- collaborative

The word cloud is the hero element.

---

# Visual Direction

Inspired by:
- Canva
- Mentimeter
- tldraw
- playful motion graphics

Avoid:
- enterprise UI
- admin dashboard feeling
- grey corporate layouts
- dense forms
- flat static screens

---

# Design Principles

Use:
- large rounded cards
- colorful gradients
- soft shadows
- subtle glow
- floating blobs
- friendly typography
- smooth microinteractions
- generous spacing
- clear hierarchy

Avoid:
- sharp corporate layouts
- too many borders
- small cramped controls
- random colors
- inconsistent spacing

---

# Word Cloud Style

The word cloud should feel:
- organic
- clustered
- layered
- dynamic
- expressive
- alive

Use:
- one large central word
- surrounding words orbiting around it
- varied word sizes
- varied rotations
- varied colors
- overlapping layers
- soft glow effects
- floating particles/confetti

Avoid:
- words in straight rows
- evenly spaced bubbles
- rigid grids
- isolated pills
- too much empty space

---

# Color Direction

Use:
- bright accent colors
- warm gradients
- playful contrast
- soft pastel backgrounds
- dark glowing presentation mode

Color should feel intentional and energetic.

Avoid:
- random color usage
- dull grey-heavy UI
- low contrast text

---

# Shape Language

Use:
- rounded corners
- soft blobs
- circular accents
- pill controls
- floating panels
- organic shapes

The UI should feel soft and tactile.

---

# Motion

Motion should feel:
- smooth
- subtle
- delightful
- responsive

Use:
- fade in
- scale on hover
- soft floating motion
- gentle word pop-in
- subtle confetti/success feedback

Avoid:
- aggressive bouncing
- distracting animations
- slow heavy transitions

---

# Typography

Typography should be:
- large
- friendly
- readable
- modern

Use clear hierarchy:
- big expressive headlines
- readable body text
- bold labels for key actions

Avoid:
- tiny labels
- dense text blocks
- overly formal typography

---

# Layout

General layout:
- mobile-first where relevant
- spacious
- clear focal point
- strong visual hierarchy

Contributor layout:
- extremely simple
- one clear question
- large input
- large submit button
- no clutter

Host/editor layout:
- creative studio feeling
- large preview area
- visual controls
- clear actions

Presentation layout:
- fullscreen friendly
- minimal UI chrome
- word cloud dominates
- works well on projector/TV

---

# Components

Components should feel:
- playful
- reusable
- polished
- consistent

Core visual components:
- Button
- Card
- Input
- Toggle
- Badge
- WordToken
- ThemePicker
- ShapePicker
- QRCard
- ExportButton
- FloatingToolbar

---

# Tailwind Usage

Use Tailwind consistently.

Prefer semantic patterns and reusable component classes.

Avoid:
- hardcoded one-off visual decisions
- inconsistent spacing
- mixing custom CSS and Tailwind randomly

Use design tokens for:
- colors
- gradients
- shadows
- radius
- spacing
- typography

---

# Success Criteria

The UI should make users feel:
- this is easy
- this is fun
- this is alive
- this will look good on screen
- this is worth sharing
