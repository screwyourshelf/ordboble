# Iteration 016.6 — Session Experience Refinement

## Goal

Polish and harden the real session experience now that the deployed live loop works end-to-end.

This iteration focuses on UX stability, graceful edge cases, and realtime resilience — without changing the core architecture.

The goal is to make Ordboble feel less like a prototype and more like a stable collaborative product.

---

# Scope

## In Scope

- Better loading states
- Session-not-found handling
- Better empty states
- Reconnect UX improvements
- Duplicate word handling improvements
- Contributor feedback polish
- Minimal realtime resilience improvements
- Friendly error messages
- Small visual polish directly tied to session experience

## Out of Scope

Do NOT:
- redesign the app
- add authentication
- add admin
- add export
- add analytics
- add rate limiting
- add Durable Objects
- add WebSockets
- add new backend architecture
- add session permissions
- add advanced moderation

---

# Requirements

## Session Loading States

Contributor and presentation screens should:
- show a proper loading state while fetching session data
- avoid flashing mock content before real content loads
- distinguish between:
  - loading
  - loaded
  - failed/not-found

Keep loading visuals lightweight and visually consistent.

---

# Session Not Found

If:
- cloud/session fetch returns 404
- session ID is invalid
- backend unreachable

Show:
- friendly session-not-found or unavailable state
- clear recovery text
- optional "Tilbake til forsiden" action

Do not crash or silently fall back to unrelated mock sessions when a route ID exists.

Important distinction:

```text
No route ID
→ prototype fallback allowed

Invalid real route ID
→ show error/not-found state
```

---

# Empty States

Presentation mode should:
- show a better "waiting for words" state
- feel alive even before first contribution
- explain how contributors join

Contributor screen should:
- better communicate successful submission
- better communicate reconnect/loading situations

---

# Duplicate Word Handling

Current behavior:
- duplicate words are allowed
- WordCluster uses stable IDs

Improve visual handling slightly:
- duplicates should not overlap awkwardly if avoidable
- repeated words should still feel intentional

Do not build a full physics/layout engine.

---

# Reconnect UX

Improve EventSource experience minimally:

- detect reconnecting state if SSE disconnects
- show subtle reconnecting indicator if useful
- reconnect automatically using EventSource default behavior
- avoid noisy console spam

Do not implement custom realtime infrastructure.

---

# Contributor Feedback Polish

Improve:
- submit feedback
- loading state
- success state
- duplicate submission feel
- disabled state clarity

Keep visual language consistent with existing design system.

---

# Technical Constraints

- Svelte 5
- TypeScript
- existing architecture only
- Cloudflare Workers + SSE remain unchanged
- no new dependencies unless clearly justified
- no redesign
- no routing rewrite
