# Iteration 012.2 — Theme and Shape System

## Goal

Create a shared frontend theme and shape system for Ordboble.

Currently, theme and shape options exist mainly inside the editor prototype. This iteration should centralize those options into shared typed models so they can later be reused by:
- editor preview
- presentation mode
- landing preview
- export UI
- future session settings

This is still frontend-only.

---

# Scope

## In Scope

- Create shared theme types
- Create shared shape types
- Create shared theme definitions
- Create shared shape definitions
- Update editor components to use the shared definitions
- Keep current editor behavior visually equivalent or slightly more consistent
- Avoid duplicate theme/shape definitions
- Keep implementation lightweight
- Verify app starts and type-checks correctly

## Out of Scope

- implement real shape layout algorithms
- implement persistence

# Architect Output

## Audit — where theme/shape data was scattered

| Data | Location | Problem |
|---|---|---|
| `EditorTheme` union | `types/editor.ts` | Misplaced — editor-scoped, not reusable |
| `EditorShape` union | `types/editor.ts` | Same |
| Theme swatch/ring | `ThemePicker.svelte` (inline) | Not reusable |
| Theme previewBg | `EditorPreview.svelte` (inline) | Duplicate theme concept |
| Theme glowColor | `EditorPreview.svelte` (inline) | Third separate theme attribute |
| Shape label/icon | `ShapePicker.svelte` (inline) | Not reusable |
| Shape label map | `EditorPreview.svelte` (inline) | Duplicate of ShapePicker data |

## Plan

1. Create `lib/types/theme.ts` — `ThemeId` union + `Theme` interface (id, label, description, swatch, ring, previewBg, glowColor)
2. Create `lib/types/shape.ts` — `ShapeId` union + `Shape` interface (id, label, icon, description)
3. Create `lib/design/themes.ts` — `themes: Theme[]` array + `getTheme(id)` helper
4. Create `lib/design/shapes.ts` — `shapes: Shape[]` array + `getShape(id)` helper
5. Update `editor.ts` — re-export `ThemeId as EditorTheme`, `ShapeId as EditorShape` (backward compatible)
6. Update `ThemePicker` — remove inline array, import from `design/themes`
7. Update `ShapePicker` — remove inline array, import from `design/shapes`
8. Update `EditorPreview` — remove three inline maps, use `$derived(getTheme/getShape)`
