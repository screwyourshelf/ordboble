# Iteration 016.3 — Session Creation Flow

## Goal

Create the first real host session creation flow.

The goal is to let a host create a real Ordboble session from the frontend and then continue to the real share URL:

```text
Landing/Create
→ POST /api/clouds
→ backend creates session in D1
→ frontend navigates to /share/:id
→ host can copy/QR/share the real join URL
```

This completes the first real host-started product flow.

---

# Scope

## In Scope

- Add frontend API client/service for creating sessions
- Add minimal host create session UI
- Connect create session UI to `POST /api/clouds`
- Navigate to `/share/:id` after successful creation
- Keep fake/prototype fallback when `env.apiBaseUrl` is empty
- Add loading and friendly error state
- Reuse existing design system and components
- Keep implementation lightweight
- Verify frontend builds and type-checks

## Out of Scope

Do NOT:
- implement authentication
- implement admin links
- implement permissions
- implement persistence outside existing D1
- change backend schema
- add routing library
- add SvelteKit
- redesign landing page completely
- implement full settings editor
- implement export
- implement localization
- add new dependencies

---

# Existing Backend Endpoint

Use existing endpoint:

```text
POST /api/clouds
```

Request body should match existing backend/shared contract.

Expected fields:
- title
- prompt
- language
- expiresAt
- maxWordsPerClient
- liveEnabled

Response should include the created session/cloud with `id`.

---

# Desired UX Flow

## Host flow

1. Host lands on app
2. Host enters or confirms:
   - title
   - prompt/question
3. Host clicks create/start
4. Button shows loading
5. On success:
   - navigate to `/share/:id`
6. On failure:
   - show friendly error message

---

# Fallback Behavior

If `env.apiBaseUrl` is empty:
- do not call backend
- create a fake/prototype session ID
- navigate to `/share/:fakeId`
- keep local/demo experience usable

This keeps Cloudflare/static prototype usable before Worker deployment.

# cloud-api.ts

Should expose a function like:

```ts
createCloud(input): Promise<...>
```

Should:
- use `env.apiBaseUrl`
- POST to `/api/clouds`
- handle backend error envelope
- return created cloud/session ID
- use fake fallback when `env.apiBaseUrl` is empty

Avoid throwing for normal API validation errors if possible.

---

# Navigation

Use simple browser navigation:

```ts
window.location.href = `/share/${id}`;
```

Do not add routing library.
