# Iteration 018.3 — Empty - Loading - Error Polish

## Goal

Refine empty, loading and error states across the real Ordboble session flow.

The app now works end-to-end. This iteration should make transitional and failure states feel intentional, stable and polished instead of temporary or abrupt.

Focus on:
- skeleton/loading states
- nicer not-found states
- better reconnect feel
- reduced layout jumps
- smoother transitions
- consistent error language

---

# Scope

## In Scope

- Improve loading states
- Improve empty states
- Improve not-found states
- Improve reconnect indicators
- Reduce layout jumps during session load
- Improve transition between mock/fallback → real session data
- Make error states visually consistent
- Reuse existing design system
- Keep implementation lightweight

## Out of Scope

Do NOT:
- redesign screens
- add new product features
- add auth/admin
- add analytics
- add routing changes
- add backend changes
- add new realtime architecture
- add new dependencies unless clearly justified
- add full i18n

# Error Language

Use friendly Norwegian microcopy for now.

Avoid:
- technical backend language
- harsh red-only error states
- scary wording
- long explanations

Prefer:
- “Vi fant ikke denne økten”
- “Kunne ikke laste økten akkurat nå”
- “Prøv igjen om litt”
- “Sjekk lenken eller gå tilbake til forsiden”

---

# Visual Direction
