# Iteration 012.1 — Shared Mock Session Model

## Goal

Create a shared mock session model for the frontend prototypes.

Currently, fake session data is spread across multiple screen components. This iteration should centralize that data so all prototype screens can reuse the same session information.

This is still frontend-only.

---

- Create shared mock session data
- Create shared session-related types if needed
- Replace duplicated fake session data in existing screens
- Keep all existing screens visually unchanged

- add backend

## Near-duplicate types
- `ShareSession` ≡ `PresentationSession` — identical shapes
- `ContributorSession` — subset; unique `helperText` field

## Plan
1. Create `lib/types/session.ts` — single `Session` type, `helperText` optional
2. Create `lib/mocks/session.ts` — canonical `mockSession` (resolves inconsistencies)
3. Replace `ShareSession`, `PresentationSession`, `ContributorSession` with `export type { Session as X }` re-exports
4. Update 4 screens to import `mockSession`

## Canonical values (resolving inconsistencies)
- `joinUrl`: `'ordboble.app/join/TEAM25'` (no scheme — display text, not href)
- `participantCount`: `24`
- `helperText`: `'Skriv ett ord om gangen — du kan legge til opptil 3 ord.'`
