# Architect Output

## Plan

Two new services. ContributorMobileScreen.handleSubmit becomes async and calls submitWords().
WordInputList gets an optional loading prop forwarded to the submit button.
SuccessConfirmation untouched. Backend route unchanged.

## Risks

- mockSession.id = 'mock-session-1' does not exist in D1. Backend returns 404 when VITE_API_BASE_URL is set and using prototype screen. Error state shows correctly. Follow-up: wire a real cloud ID.
- localStorage unavailable in private mode — handled with try/catch fallback to crypto.randomUUID().

# Developer Output

## client-id.ts

- localStorage key: ordboble.clientId
- Creates UUID if missing, reuses on next visit
- Falls back to in-request random UUID if localStorage throws

## word-api.ts

- If apiBaseUrl empty: returns {ok:true, accepted:words} (fake path)
- If apiBaseUrl set: POST /api/clouds/:id/words with clientId + words
- Extracts error.message from backend error envelope
- If all words are duplicates (empty data array): shows submitted words as success
- Never throws — returns typed result union

## ContributorMobileScreen

- handleSubmit now async
- loading = true while awaiting
- On ok: sets submittedWords = result.accepted, submitted = true
- On error: sets error = result.error, stays on input screen
- error clears on next submit attempt

## WordInputList

- New optional loading prop (default false)
- Button disabled when loading or no valid words
- Button label: "Sender inn..." while loading

```
POST /api/clouds/:id/words  {"clientId":"...","words":["tillit","åpenhet","humor"]}
-> 201, 3 words accepted and stored

Same client, same words again
-> 400 "Max words per client (3) already reached"
-> word-api.ts extracts message, ContributorMobileScreen shows error

-> 404 "Cloud not found"
-> word-api.ts extracts message, ContributorMobileScreen shows error
```

All cases handled gracefully.

# Reviewer Output

**Simplicity:** Two small services, one async handler, one optional prop. No global state.

**Fallback:** Empty apiBaseUrl returns fake success immediately — prototype unchanged.

**Error UX:** Backend error.message surfaced in Norwegian-friendly context. Clears on retry.

**Duplicates:** Empty data array falls back to showing submitted words — user sees success rather than confusion.

**Scope:** Clean. No backend changes, no routing, no redesign.
