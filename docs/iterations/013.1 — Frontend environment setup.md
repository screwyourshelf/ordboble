# Iteration 013.1 — Frontend Environment Setup

## Goal

Establish a minimal frontend environment configuration strategy.

The goal is to prepare the app for different frontend environments without overengineering.

This iteration should support:
- local development
- Cloudflare Pages preview/production
- future API base URL configuration

This is frontend-only.

---

# Scope

## In Scope

- Create typed frontend environment access
- Add safe default values for local/prototype mode
- Document supported environment variables
- Prepare for future API base URL
- Prepare for future default locale/domain behavior
- Keep implementation minimal
- Verify app starts and type-checks correctly

## Out of Scope

Do NOT:
- add backend
- add Cloudflare Workers
- add D1
- add SSE
- add authentication
- add localization implementation
- add routing
- add persistence
- add secrets
- add complex environment framework
- change UI/UX

# Environment Variables

Support these variables:

```text
VITE_APP_NAME=Ordboble
VITE_DEFAULT_LOCALE=nb
VITE_API_BASE_URL=
```

For now:
- `VITE_API_BASE_URL` may be empty
- no API calls should be added
- locale is only stored/configured, not used for full localization

---

# Requirements

## env.ts

Create a small typed config object.

It should expose:
- appName
- defaultLocale
- apiBaseUrl

It should:
- read from `import.meta.env`
- provide safe defaults
- not throw if values are missing

Example behavior:
- appName defaults to `Ordboble`
- defaultLocale defaults to `nb`
- apiBaseUrl defaults to empty string

---

# Documentation

`.env.example` should include:
- supported variables
- short comments
- no secrets
