# Iteration 013.2 — Cloudflare Worker Foundation

## Goal

Create the first minimal Cloudflare Worker foundation.

The goal is to prepare for future backend/API work without implementing real product functionality yet.

This iteration should establish:
- Worker app structure
- local Worker development
- minimal health endpoint
- minimal API response pattern
- deployment readiness

---

# Scope

## In Scope

- Create `apps/worker`
- Add minimal Cloudflare Worker project
- Add Wrangler configuration
- Add TypeScript setup for Worker
- Add health endpoint
- Add minimal JSON response helper if useful
- Verify Worker can run locally
- Verify Worker can build/type-check
- Keep implementation minimal

## Out of Scope

Do NOT:
- add D1
- add database migrations
- add SSE
- add realtime
- add authentication
- add session APIs
- add word submission APIs
- connect frontend to Worker
- add shared API contracts yet
- add complex routing framework
- add validation library
- add new product behavior

---

# Expected Structure

Create:

```text
apps/
  worker/
    src/
      index.ts
    package.json
    tsconfig.json
    wrangler.jsonc
```

---

# API Endpoints

## Health

```text
GET /health
```

Expected response:

```json
{
  "ok": true,
  "service": "ordboble-worker"
}
```

## Root

```text
GET /
```

Expected response:

```json
{
  "name": "ordboble-worker",
  "status": "ok"
}
```

---

# Requirements

## Worker

The Worker should:
- use TypeScript
- return JSON responses
- handle unknown paths with `404`
- avoid product logic
- avoid database bindings
- avoid environment complexity

---

# Wrangler

Wrangler config should:
- use project name `ordboble-worker`
- point to `src/index.ts`
- use compatibility date
- not include D1 bindings yet

---

# Technical Constraints

- Cloudflare Workers
- TypeScript
- minimal dependencies
- no backend framework unless clearly justified
- no D1
- no SSE
- no auth
- no frontend integration yet
