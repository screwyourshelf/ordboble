# Iteration 012.3 — Export UI Foundation

## Goal

Create a shared frontend export UI foundation for Ordboble.

Export is not implemented yet. This iteration should establish reusable export types, options and UI components that can later support:
- PNG
- SVG
- PDF
- Print

This is frontend-only and prototype-only.

---

# Scope

## In Scope

- Create shared export types
- Create shared export option definitions
- Create reusable export UI components
- Update editor export panel to use shared export definitions
- Keep all export actions as prototype/disabled/coming soon
- Keep implementation lightweight
- Verify app starts and type-checks correctly

## Out of Scope

Do NOT:
- implement real export
- implement print
- implement file downloads
- implement canvas/SVG rendering
- implement PDF generation
- add backend
- add realtime
- add routing
- add persistence
- add new dependencies

# Architect Output

## Audit — current export code

`ExportPanel.svelte` contains a 4-item inline array `{ label, icon }` only. No shared type, no id, no description, no enabled flag. Completely self-contained with no reuse potential.

## Plan

1. Create `lib/types/export.ts` — `ExportFormatId` union + `ExportOption` interface
2. Create `lib/design/export-options.ts` — `exportOptions[]` (all `enabled: false`)
3. Create `lib/components/export/ExportActionButton.svelte` — single option, disabled state
4. Create `lib/components/export/ExportActions.svelte` — renders all options via grid
5. Update `ExportPanel.svelte` — replace inline array + buttons with `<ExportActions />`
