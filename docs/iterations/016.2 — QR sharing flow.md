# Iteration 016.2 — QR Sharing Flow

## Goal

Add the first real QR sharing flow.

The goal is to make the share screen produce a real join URL and QR code that points contributors to the correct session URL:

```text
/join/:id
```

This makes the host share flow usable for real devices.

---

# Scope

## In Scope

- Generate a real join URL from the current session ID
- Replace fake QR placeholder with a real QR code if feasible
- Keep a graceful fallback QR-style placeholder if real QR generation is not practical
- Update share screen to use route/session ID
- Update copy link behavior to copy the real join URL
- Keep share UI visually consistent
- Keep implementation frontend-only
- Verify frontend builds and type-checks

## Out of Scope

Do NOT:
- implement backend changes
- implement session creation UX
- implement authentication
- implement admin links
- implement permissions
- implement export
- redesign share screen
- add complex QR customization
- add paid/heavy dependencies

---

# Requirements

## Join URL

Generate join URL from:

```text
window.location.origin + /join/:sessionId
```

If no real session ID exists:
- use mock session fallback
- keep prototype behavior working

---

# QR Code

Preferred:
- use a small lightweight QR dependency only if needed

Acceptable:
- simple frontend QR generation
- SVG/canvas QR output
- no backend

Avoid:
- large/heavy QR libraries
- remote QR services
- backend QR generation

If adding a dependency:
- keep it minimal
- document why it is needed

---

# Share Screen

Update share screen so:
- QR points to the real join URL
- displayed link is the real join URL
- copy button copies the real join URL
- session ID comes from route ID if present
- fallback still works in prototype mode

---

# Technical Constraints

- Svelte 5
- TypeScript
- frontend-only
- no backend changes
- no routing changes beyond using existing route session ID
- no UI redesign
- keep dependency footprint small

---

# Verification

Verify:
- `/share/test-id` displays a join link ending in `/join/test-id`
- copy button copies the correct URL
- QR represents the same URL
- frontend type-checks
- frontend builds
- fallback still works without route session ID
