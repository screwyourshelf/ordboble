# Iteration 018.1 — Presentation Visual Polish

## Goal

Refine the visual feel of presentation mode now that the collaborative flow and customization system are stable.

This iteration focuses entirely on:
- composition
- atmosphere
- readability
- visual balance
- motion quality

The goal is to make presentation mode feel intentional, immersive and polished during real usage:
- workshops
- classrooms
- conferences
- facilitation
- retrospectives

without redesigning the product.

---

# Scope

## In Scope

- Better cloud composition
- Better word spacing/balance
- Refined motion intensity
- Reduced visual noise
- Theme-specific atmosphere tuning
- Better fullscreen presentation feel
- Improved typography hierarchy
- Better empty-state composition
- Subtle depth and layering improvements
- Better visual rhythm

## Out of Scope

Do NOT:
- redesign app architecture
- rebuild layout engine
- add physics simulation
- add 3D effects
- add particle systems
- add export
- add new features
- add auth/admin
- add new realtime architecture

---

# Requirements

## Better Cloud Balance

Current issue:
- words can cluster awkwardly
- composition sometimes feels random/noisy
- center weighting inconsistent

Improve:
- spacing distribution
- visual balance
- breathing room
- hierarchy between words

Keep implementation lightweight.

Acceptable:
- refined grid/ring placement
- weighted spacing
- soft collision avoidance
- subtle scale balancing

Do NOT build a heavy physics engine.

---

# Motion Refinement

Current motion can feel:
- slightly noisy
- overly glowy
- too uniformly animated

Improve:
- calmer motion rhythm
- varied animation intensity
- reduced simultaneous motion
- more premium feel

Themes should influence motion subtly.

Examples:
- Calm → slower, softer
- Playful → more energetic
- Dark → more atmospheric

Do NOT introduce complex animation systems.

---

# Reduced Visual Noise

Reduce:
- excessive glow stacking
- overlapping visual hotspots
- over-saturated combinations
- too many competing accents

Improve:
- focus
- readability
- visual hierarchy

Presentation should feel:
- easier to read from distance
- more intentional
- more professional

---

# Theme Atmosphere

Refine themes so they feel more distinct.

## Playful
- vibrant
- energetic
- event/workshop feel

## Calm
- subdued
- spacious
- presentation/classroom feel

Theme differences may affect:
- background gradients
- orb opacity
- animation intensity
- glow amount
- word spacing
- typography emphasis

Keep implementation token/preset-based.

---

# Better Fullscreen Feel

When chrome hidden/fullscreen:
- presentation should feel immersive
- composition should scale gracefully
- word cloud should occupy space more intentionally
- avoid awkward empty corners

May include:
- better max widths
- adaptive spacing
- responsive cloud scaling
- fullscreen-specific atmosphere tweaks

---

# Typography & Readability

Improve:
- title/prompt hierarchy
- readability at distance
- contrast
- cloud readability over background

Avoid:
- tiny UI text
- low contrast overlays
- excessive blur/glow behind words

---

# Empty State Polish

Presentation with zero words should:
- feel intentional
- not feel broken or abandoned
- gently guide participation

Keep subtle and elegant.

---

# Technical Constraints

- Svelte 5
- TypeScript
- existing architecture only
- no new dependencies unless clearly justified
- keep rendering lightweight
- maintain mobile safety
- maintain deployed realtime behavior

---

# Expected Files

Likely:
- PresentationModeScreen.svelte
- WordCluster.svelte
- word-cloud positioning helpers
- theme tokens/config
- motion utilities/styles
- shared UI atoms if needed

Only modify files required for visual polish.

---

# Verification

Verify:
- presentation feels calmer and more balanced
- calm/playful themes clearly differ
- fullscreen feels immersive
- readability improved
- no realtime regressions
- no major FPS/render regressions
- frontend builds/type-checks
- deployed presentation still works
