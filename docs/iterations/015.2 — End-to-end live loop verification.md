# Architect Output

## Problem

Both ContributorMobileScreen and PresentationModeScreen used mockSession.id ('mock-session-1').
That ID does not exist in D1, so the full loop could not be verified.
Neither screen was using the same cloud ID for submit and SSE subscription.

## Solution

Add VITE_CLOUD_ID env variable.
When set: both screens use it as the cloud ID.
When empty: both fall back to mockSession.id (prototype mode, unchanged).

Change is 6 lines across 4 files. No routing, no session loading, no UI changes.

## Risks

- VITE_CLOUD_ID is dev-only. Documented clearly as temporary. Real solution is routing with cloud ID in URL.
- If VITE_CLOUD_ID is set but VITE_API_BASE_URL is empty: fake path still runs correctly.

## E2E Test Flow

```
# 1. Start worker
pnpm exec wrangler dev --port 8787

# 2. Create cloud
POST /api/clouds {"title":"Team workshop","language":"no","maxWordsPerClient":10}
-> cloudId: 7dcbdf7f-c166-4d69-bf81-84420c3b6580

# 3. Set .env.local
VITE_API_BASE_URL=http://127.0.0.1:8787
VITE_CLOUD_ID=7dcbdf7f-c166-4d69-bf81-84420c3b6580

# 4. Open frontend
- presentation screen: subscribes to SSE on cloudId
- contributor screen: submits to cloudId

# 5. Submit words via POST /api/clouds/:id/words
-> 201 Created, words stored in D1

# 6. SSE stream open on same cloud
-> GET /api/clouds/:id/events 200 OK
-> word_added event emitted within 2s poll cycle
```
