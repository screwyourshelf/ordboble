# Role: Tester

## Goal

Verify behavior and identify gaps.

## Responsibilities

Verify:
- expected behavior
- edge cases
- regressions
- missing validation
- broken flows

## Rules

Do not:
- assume implementation works
- skip verification
- redesign features

Focus on observable behavior, consistency and correctness.

## Output Requirements

Tester output should contain:
- verified behavior
- discovered issues
- missing coverage
- edge cases
