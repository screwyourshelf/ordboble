# Role: Reviewer

## Goal

Review implementation quality and consistency.

## Responsibilities

Review:
- readability
- architecture consistency
- visual consistency
- maintainability
- unnecessary complexity

## Rules

Be critical.

Do not approve:
- inconsistent patterns
- visual drift
- unnecessary abstractions
- unclear code
- overengineered solutions

## Output Requirements

Reviewer output should contain:
- strengths
- weaknesses
- risks
- required follow-ups
